# BriefCase 🚀

BriefCase is an elite corporate operations and voice-automation workspace agent designed to bridge the gap between advanced artificial intelligence architectures and fluid human productivity. By converting priority inbox traffic, custom rule matrices, and calendar states into dynamic spoken executive briefs, BriefCase empowers professionals to command their workflow seamlessly without ever looking at a screen.

The live web application is securely deployed and globally accessible at: **[briefcase-nqsy.onrender.com](https://briefcase-nqsy.onrender.com/)**

---

## ✨ Features

*   **🎙️ Smart Executive Audio Briefings:** Dynamically synthesizes a high-fidelity real-time spoken digest matching structural environmental metrics, device clock metadata, localized weather data, and custom-styled speech parameters. The engine enforces a strict protocol ensuring briefings always begin natively with the verbal phrase *"Good Morning, Boss."*
*   **⏳ History-Aware Briefing Logs (No Re-reading):** Solves the recurring speech bottleneck by integrating an atomic write tracking table. The database stores processed message identifiers inside a dedicated cache layer to screen out historical reads, ensuring that hitting the briefing trigger only announces fresh, unread priority items.
*   **⚡ 30-Second Fast Auto-Star Ring Fence:** Implements an automated high-frequency background worker polling system that wakes up every 30 seconds to run inbox sweeps against user-defined sender metrics. It automatically flags qualifying incoming records with a Gmail priority star and provisions custom organizational labels (e.g., *from school*, *from work*) completely behind the scenes.
*   **✉️ Conversational Command Control Terminal:** An interactive natural language processing layer capable of executing full operational tasks from a single prompt. It intelligently parses human commands to either add priority targets to the matrix rule grid, star a recent unstarred message, look up historical context, or draft corporate outbox deliveries.
*   **🔒 Structural Signature Lock & Custom Watermarking:** Protects formal outbox text composition layouts from AI hallucinations. The agent splits the body content from the closure step, permanently forcing professional sign-offs (`Best regards,` or `Warm Regards,`) to sit stacked with a clean newline break directly above the user's verified profile account name. Concludes with a beautifully muted italics tracking watermark.
*   **🛠️ Resilient 503 Auto-Healing & Sockets Handling:** Built to handle heavy public developer tier traffic bottlenecks cleanly. Implements an automated Exponential Backoff loop around generative API targets to intercept `503 Service Unavailable` statuses, while integrating self-healing wrapper handlers around Edge neural speech requests to prevent silent frontend audio player dropouts.

## 🛠️ Tech Stack & Architecture

*   **Runtime Environment & Core Backend Node Engine:** Express.js framework managing unified JSON API endpoints, routing states, static asset streaming, and automatic browser window execution mappings.
*   **Generative AI Orchestration Layer:** Powered by official `@google/genai` structures utilizing the high-speed `gemini-2.5-flash` foundational model for rapid multi-intent classification, structural schema formatting, and conversational reasoning.
*   **Vocal Synthesis Matrix:** Driven by neural text-to-speech protocols (`msedge-tts`) utilizing clear, professional `en-US-AriaNeural` vocal data formats running on high-bitrate single-channel parameters.
*   **Relational Cloud Host:** Managed securely via Supabase leveraging PostgreSQL data layers to track native user auth access states, dynamic rules configurations, and historic message lookup filter vectors.
*   **OAuth Authorization Integrations:** Connected via official Google API clients (`googleapis`) allowing the secure cloud runtime to handle modifications of live Gmail scopes, custom labels, and primary calendar event sequences.

## 📦 Database Schema Setup

To support the automation routing matrix, rules engine, and history log delta protections, configure the following relational tables directly inside your Supabase SQL Workspace console:

```sql
-- 1. Automated Star Rules Vector Grid
create table public.auto_star_rules (
    id bigint generated by default as identity primary key,
    user_email text not null,
    target_sender text not null,
    custom_label text not null,
    created_at timestamp with time zone default timezone('utc'::text, now()) not null,
    
    unique(user_email, target_sender)
);

-- 2. Briefing Duplication Filter Log Cache
create table public.read_briefing_logs (
    id bigint generated by default as identity primary key,
    user_email text references public.users(email) on delete cascade,
    message_id text not null,
    read_at timestamp with time zone default timezone('utc'::text, now()) not null,
    
    unique(user_email, message_id)
);
```

## ⚠️ Crucial Access Configuration Note: To enable your local server environment or your production cloud instances to instantly save rules or check read logs without experiencing permission handshaking hangs, remember to safely Disable Row-Level Security (RLS) on these specific tables via the Supabase Table Editor panel dashboard.

## 🚀 Local Installation & Execution
Follow these step-by-step instructions to boot up and test the active developer environment on your local machine:

* **Clone the Repository:**

```Bash
   git clone https://github.com/aish-2306/BriefCase.git
   cd BriefCase
```
* **Install Local Node Modules:**

```Bash
   npm install
```
* **Configure Your Environment Parameters:**
Create a secure file named .env directly in your root project folder (at the same level as server.js) and populate it with your private cloud platform keys:

* **Code snippet**
   ```
   SUPABASE_URL=https://your_project_id.supabase.co
   SUPABASE_SERVICE_ROLE_KEY=your_secret_supabase_service_role_token
   GEMINI_API_KEY=your_google_ai_studio_api_key
   GOOGLE_CLIENT_ID=your_gcp_oauth_client_id.apps.googleusercontent.com
   GOOGLE_CLIENT_SECRET=your_gcp_oauth_client_secret
  ```

* **Launch the Core Engine Server:**

```Bash
   npm start
```
The local application instance will turn active on port 5000 and automatically spawn a browser layout pointing directly to http://localhost:5000.
